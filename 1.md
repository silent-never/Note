# E34 FSAE Vehicle Parameter Identification

  

从四电机独立驱动大学生方程式赛车 (E34) 的跑动数据中自动识别动力学参数、悬架参数、质心位置和气动力曲线。

  

## 原理

  

基于车辆动力学模型，从传感器数据反推车辆物理参数：


| 类别 | 参数 | 方法 |

|------|------|------|

| 质心 | CG 纵向位置 (x) | 静载分配 Front:Rear = 7:9 |

| 质心 | CG 高度 (z) | 制动俯仰梯度 + 悬架固有频率 → 载荷转移反推 |

| 操稳 | 不足转向梯度 K_us | 自行车模型稳态过弯回归: δ − L·ω/v = K_us·a_y |

| 操稳 | 轮胎附着系数 μ | GG 图包络 (max √(a_x²+a_y²)/g) |

| 悬架 | 固有频率 f_n | 差动信号 + 去趋势 + 速度自相关 |

| 动力 | 最大功率 / 能耗 | P = Σ(T·ω), E = ∫P·dt |

| 动力 | 前后功率分配 | 前/后轴功率占比 |

| 气动 | 阻力 vs 速度 | 滑行段回归, 自动选最优函数形式 |

| 气动 | 前后轴下压力 vs 速度 | 直道悬架压缩 → MR×s×KW 换算 |

## CSV 数据格式

  

AiM 数据记录器导出的 CSV，按列名匹配（与列顺序无关）：

  

```

Time, GPS Speed, GPS Slope, GPS Heading, GPS Gyro, GPS Radius,

Steer, Bat Voltage, current, a-y, a-x,

LFSuspensionWir, RFSuspensionWir, HeaveWir, RollWir,

Motor Speed RR/LR/RF/LF, Motor Torque RR/LR/RF/LF

```

  

可选列 `ST` — 车手状态（3 = 就绪），自动过滤非 ST=3 的数据。

  

## 使用

  

```bash

python3 param_identification.py <csv_file>

```

  

零外部依赖，仅使用 Python 标准库，兼容 Windows / Linux。

  

## 关键算法

  

### 前悬架死区处理

  

前悬线位移传感器在伸长方向（共同模式）存在机械死区：

- **频率检测**：使用差动信号 LF−RF 避死区；同时检测位置和速度信号

- **俯仰梯度**：优先使用制动方向数据 (a_x < −1.5 m/s²)，此时前悬压缩无死区

  

### 加速度计零偏校正

  

从静止时段 (GPS Speed < 0.3 m/s) 自动计算 a_x/a_y 零偏并扣除。

  

### 气动力拟合

  

自动尝试 `a + b·v²` 和 `a + b·v + c·v²`，选调整 R² 最优者。下压力通过悬架运动学换算：

  

```

F_axle = 2 × MR × s_sensor[mm] × KW     (两侧合计)

KW = k_spring × MR²

```

  

阻力通过滑行段回归：`F_drag = -m·a_x` (平坦路面假设，含传动系拖滞)。

## 局限性

  

1. **C_d·A / C_rr 含传动系拖滞** — 不是纯气动/轮胎参数

2. **K_us 不确定性大** — 20 Hz GPS 偏航角速度噪声大，R² < 0.1

3. **弹簧刚度 ±30%** — 频率检测分辨率受限于 20 Hz 采样

4. **CG 高度依赖俯仰梯度 + 频率** — anti-dive/anti-squat 几何会分担部分载荷转移

5. **下压力依赖 MR/KW 准确性** — 运动比和轮心刚度来自 Excel 设计值
