1. 拆分耗时

当前 `time_elapsed` 测量的是整个 `solve()`，快速解耦法包含了 B′/B″构造和分解。建议拆成：

- 矩阵构造耗时
- 矩阵分解耗时
- 迭代回代耗时
- 总求解耗时
- 平均每次迭代耗时

这能直接证明快速解耦法的优势来自“只分解一次并反复回代”，也能判断时间花在矩阵构造还是迭代上。

2. 交替执行 NR 和快速解耦法

目前在 [solver_comparison.py (line 184)](/D:/Data/Git/Power-Flow/src/analysis/solver_comparison.py:184) 中，是先连续运行全部 NR，再连续运行全部快速解耦法。这可能受到 CPU 升频、缓存、后台程序等影响。

更公平的顺序应当类似：

```
第1轮：NR → FDLF
第2轮：FDLF → NR
第3轮：NR → FDLF
...
```

或者每轮随机顺序，并保持两个算法使用相同的初始电网。


4. 增加“残差—时间”曲线

当前残差曲线横坐标是迭代次数，但一次 NR 迭代和一次 FDLF 迭代的成本不同。建议再画：

```
横坐标：累计运行时间
纵坐标：最大功率不平衡量
```

它能回答一个更实际的问题：算法运行到 1 ms、10 ms 时，误差分别降到了多少。

5. 更完整地测量收敛特性

除了观测收敛阶，还可以记录：

- 每步误差缩减比：`e(k+1) / e(k)`
- 每毫秒误差下降量
- 达到 `1e-2、1e-4、1e-6、1e-8` 所需时间
- 最后几步观测阶的中位数，而不只使用最后一个有效值

最后一个观测阶容易受机器精度和收敛阈值影响。使用渐近阶段多个观测阶的中位数会更加稳定。

6. 不同精度要求下的比较

分别测试：

```
1e-4、1e-6、1e-8、1e-10
```

快速解耦法在较宽松精度下可能非常高效，而要求很高精度时迭代次数可能明显增加。这比只固定 `1e-6` 更全面。

7. 增加鲁棒性测试

对于每个 IEEE 系统，可以测试：

- 平坦启动
- 数据文件给定初值
- 随机扰动初值
- 不同负荷倍率
- 接近鼻点的重载状态
- 较高线路 R/X 比

统计：

- 收敛成功率
- 最大可收敛负荷倍率
- 平均迭代次数
- 失败时的最终残差
- 振荡、停滞还是数值发散

这一项很重要，因为标准 IEEE 输电系统通常天然有利于快速解耦法，不能只依靠正常工况判断其鲁棒性。

8. 测量规模增长趋势

不仅按节点数，还应记录：

- PQ节点数
- PV节点数
- 支路数
- 雅可比矩阵维度
- 矩阵非零元数量
- 每次迭代耗时

然后绘制：

```
系统规模/非零元数 → 运行时间
```

节点数相同的两个系统，网络稀疏程度不同，计算量可能差别很大。

9. 测量内存和矩阵实现差异

当前实现主要采用稠密 NumPy 矩阵。对于较大系统，可以进一步比较：

- 稠密矩阵内存
- 稀疏矩阵内存
- 稠密 LU 耗时
- 稀疏 LU 耗时
- SciPy LU 与 NumPy 逆矩阵后备方案

尤其不建议把“缓存逆矩阵”当作最终高性能方案；稀疏 LU 分解通常会更加稳定，也更符合大型电力系统计算方式。

最推荐优先实现的是：

1. 分阶段计时；
2. NR/FDLF 交替运行；
3. 残差—累计时间曲线；
4. 不同负荷倍率和不同精度下的收敛成功率。
