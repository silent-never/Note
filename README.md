# EF26_Model 电动方程式赛车 - 整车设计与模型仓库
版本状态：2026赛季设计总装-未审查各子系统 | 更新日期：2026.06.27

## 📖 项目简介
本仓库为 EF26 赛季电动方程式赛车的**全车结构三维建模**（CATIA V5 工程文件）的版本控制中心。

依托该仓库，我们实现了整车各子系统的协同设计、迭代追踪与数据同步，涵盖空气动力学、底盘、传动、高压/低压电气及人机交互等核心模块。

## 🗂️ 目录结构说明
### 顶层架构
EF26_Model/
├── EF26_Sys_Aerodynamic            # 空气动力学系统（前翼、侧扩、尾翼）
├── EF26_Sys_BrakeSystem            # 制动系统（）
├── EF26_Sys_Cooling                # 冷却系统（散热器、水泵、冷却管路布局）
├── EF26_Sys_Coordinator            # 整车坐标系基准文件（全局定位母版）
├── EF26_Sys_Electric               # 电气系统（电池箱、电机控制器、高压线束、低压线束）
├── EF26_Sys_Ergonomics             # 人机工程（仪表板、方向盘、座椅）
├── EF26_Sys_Monocoque              # 单体壳（复合材料车身结构、铺层设计）
├── EF26_Sys_SteeringSystem         # 转向系统（）
└── EF26_Sys_Suspension&Shocks      # 悬架与减震系统（）

### 协同设计规范
1. 基础命名通则
前缀强制：文件名最前必须包含 EF26（或通用前缀 AF&EF26），用于区分赛季及车型（电车/无人车）。
全英文命名：仅可使用英文单词，每个独立单词首字母大写，单词间不加空格（如 HighVoltage）。
禁用后缀：严禁在文件名末尾添加个人姓名、时间戳或版本号（版本信息由 Git 提交记录管理）。
不可变性：总装模型一旦汇总定稿，其主文件名永久锁定，不得更改。

2. 层级命名
系统文件夹（Sys）	EF26_Sys_系统名	EF26_Sys_Electric
